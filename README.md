# SocioMol Arm Builder

SocioMol Arm Builder is a lightweight local web app for picking and recording two reference points per arm before downstream RELION center shifting and SocioMol STAR generation.

This MVP does not fit a model to a map. It assumes any optional PDB/mmCIF model has already been fitted externally and only uses that model as a visual overlay.

## MVP Scope

- Load one reference STA map in `.mrc` format.
- Accept voxel size input and show map metadata.
- Optionally load an externally fitted `.pdb`, `.cif`, or `.mmcif` model.
- Display the map and optional model in a 3D viewer.
- Use the map box center as the exported coordinate origin.
- Define each arm from exactly two recorded points: an **Anchor** (Exit Point) and a **Guide Point**.
- Set the tangent direction per arm (inward or outward) to indicate how SocioMol should orient the arm vector.
- Show each arm's saved point coordinates.
- Rename, edit, nudge, and delete arms in the default UI.
- Export `arm_geometry.json`.

## Product Boundary

The app is only responsible for recording centered Angstrom arm coordinates. It does not:

- fit a model to a map;
- generate RELION center-shifted STAR files directly;
- implement IMOD as a user-facing input mode;
- introduce a reference-voxel coordinate mode;
- compute or export derived arm geometry.

## Architecture

The MVP is intentionally small:

- `app.py`: serves the static frontend with Flask.
- `static/viewer.js`: owns the 3D viewer, map/model loading, point capture, and simple two-point overlay rendering.
- `static/state.js`: keeps arm definitions, draft state, display toggles, and preset hooks.
- `static/export.js`: builds and downloads `arm_geometry.json` plus a hidden preview marker helper export.
- `static/api.js`: handles lightweight client-side MRC header metadata parsing.
- `static/presets.js`: holds future-facing preset and landmark hooks that are hidden in the simplified default UI.

## How To Run Locally

```bash
cd sociomol_arm_builder
pip install -r requirements.txt
python app.py
```

Then open:

```text
http://127.0.0.1:5000
```

The frontend uses NGL from a CDN to avoid a Node build step. For fully offline lab use, download `ngl.js` locally and replace the CDN script tag in `static/index.html`.

## Simplified Default Layout

The default interface is optimized around one main workflow:

1. Load the reference map and optional fitted model from the compact top bar.
2. Inspect the dominant 3D viewport.
3. Capture or type the Anchor and Guide Point coordinates.
4. Set the tangent direction (Guide Point → Anchor for inward picks, Anchor → Guide Point for outward picks).
5. Refine coordinates numerically, including nudge.
6. Save arms in the side panel.
7. Export `arm_geometry.json`.

The main side panel intentionally keeps only:

- `Arm Editor`
- `Defined Arms`
- `Export`

Validation references, preset hooks, preview marker export, and a few secondary helper controls remain in code but are hidden from the default UI in this pass.

## Coordinate Convention

- Exported coordinates are centered Angstrom coordinates.
- Internal/export origin is the reference map box center.
- Axis order is `[x, y, z]`.
- A coordinate such as `[-24, -12, 28]` means `x=-24 Angstrom`, `y=-12 Angstrom`, `z=28 Angstrom` relative to the map box center.
- These coordinates are the values intended to correspond to RELION `--center_X`, `--center_Y`, and `--center_Z`.

## Refined Coordinate Editing Workflow

1. Load a map and optional externally fitted model.
2. Either pick the Anchor and Guide Point in the viewer or type known centered Angstrom coordinates directly.
3. Use the numeric coordinate boxes for precise editing.
4. Use the nudge controls to move the Anchor or Guide Point by a configurable Angstrom step along X, Y, or Z.
5. Choose the tangent direction for each arm.
6. Save the arm, then export `arm_geometry.json`.

Numeric input is intentionally tolerant while editing. Partial values are not treated as final coordinates until the field contains a complete valid X/Y/Z triplet or the user saves.

## Model Display Options

The optional fitted model can be displayed as:

- `Cartoon + sidechains`
- `Cartoon`

These are visualization options only. The app does not move, refit, or modify the model.

## `arm_geometry.json` Structure

The exported file is directly compatible with `sociomol preprocess --model-json`. It includes arm point coordinates, tangent direction, and source and coordinate metadata:

```json
{
  "name": "custom_model",
  "description": "Generated by SocioMol Arm Builder from reference_map.mrc",
  "schema_version": "0.2.0",
  "coordinate_system": {
    "origin": "map_box_center",
    "units": "angstrom",
    "axis_order": ["x", "y", "z"]
  },
  "source": {
    "map_file": "reference_map.mrc",
    "model_file": "fit_model.cif",
    "grid_size_voxels": [160, 160, 160],
    "voxel_size_angstrom": [2.12, 2.12, 2.12],
    "box_size_angstrom": [339.2, 339.2, 339.2]
  },
  "arms": [
    {
      "anchor": [-24.0, -12.0, 28.0],
      "direction_point": [-36.0, -8.0, 0.0],
      "tangent": "direction_point_to_anchor"
    }
  ]
}
```

## Field Definitions

- `anchor`: the Exit Point of the arm (formerly `point1_xyz`), relative to the map box center, in Angstrom.
- `direction_point`: the Guide Point used to define the arm's tangent vector (formerly `point2_xyz`), in Angstrom.
- `tangent`: tangent direction convention.
  - `"direction_point_to_anchor"` — arm vector points from Guide Point toward Anchor (use when you picked the Guide Point inward from the particle).
  - `"anchor_to_direction_point"` — arm vector points from Anchor toward Guide Point (use when you picked the Guide Point outward).

## Presets And Landmark Hooks

The MVP keeps placeholders for predefined anchor points, biological landmarks, and future nucleosome/ribosome presets. These hooks are hidden from the simplified default UI, and the app does not invent landmark coordinates in the first version.
